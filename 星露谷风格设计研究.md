# 星露谷风格设计研究

## 一、视觉设计特点

### 1.1 像素美学
- **分辨率**: 320×240像素（16-bit时代风格）
- **调色板**: 每个元素仅用6-8种颜色
- **禁用纯色**: 不使用纯黑(#000)或纯白(#FFF)
- **边缘处理**: 轮廓线是最暗的颜色

### 1.2 色彩系统
| 类型 | 用途 | 示例 |
|------|------|------|
| 功能色 | 红色=危险/重要，绿色=安全/可交互 | ⚠️ ❌ ✅ |
| 场景色 | 早晨暖色调，夜晚冷色调 | 🌅 🌙 |
| 状态色 | 能量充足蓝色，低能量红色 | 💙 ❤️ |
| 季节色 | 春樱粉、夏绿、秋橙、冬蓝 | 🌸 🌿 🍂 ❄️ |

### 1.3 阴影技法
- **抖动(Dithering)**: 曲面阴影、部分阴影使用交叉网格
- **实心阴影**: 投影、整个面在阴影中使用实心

## 二、交互设计特点

### 2.1 反馈机制
| 交互场景 | 反馈方式 |
|----------|----------|
| 物品拾取 | 图标弹跳 + 数值浮动 |
| 任务完成 | 提示框渐入 + 音效 |
| 技能升级 | 光芒四射效果 + 等级提升动画 |
| 对话选择 | 选项高亮 + 轻微缩放 |
| 按钮交互 | 悬停亮度+10%，按下位移1px+亮度-10% |

### 2.2 布局架构
```
┌─────────────────────────────────────┐
│          状态信息区（顶部）            │
│      生命值 | 能量值 | 时间 | 天气      │

│                                     │
│                                     │
│          核心操作区（中央）            │
│           游戏主视图                  │
│                                     │
│                                     │
├─────────────────────────────────────┤
│          快捷工具栏（底部）            │
```

## 三、社交系统特点

### 3.1 NPC系统
- **专属作息**: 每个NPC有自己的日程表
- **隐藏故事**: 通过好感度解锁
- **动态关系**: 赠礼、对话影响关系
- **好感度显示**: ❤️❤️❤️❤️❤️（最多10颗心）

### 3.2 任务系统
- **不强迫**: 任务栏不会催促完成
- **自动结算**: 错过的任务次日自动结算奖励
- **渐进引导**: 从简单到复杂逐步引入

## 四、五大设计原则

### 1. 一致性优先
- 所有按钮采用相同像素边框
- 图标风格、大小、线条粗细统一

### 2. 反馈即时化
- 每个操作都有视觉/听觉反馈
- 状态变化清晰可见

### 3. 层次可视化
- 通过空间布局区分信息重要性
- 使用大小、颜色、位置引导注意力

### 4. 情感化设计
- 节日主题界面
- 季节变化影响视觉
- 温馨治愈的氛围

### 5. 渐进式引导
- 从基础开始，逐步增加复杂度
- 视觉提示引导学习新元素
