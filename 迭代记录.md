## 2026-05-02 - Day 17 迭代

### 迭代内容

**目标**：API 安全增强与请求优化 - 基于 Day 17 学习的 API 网关设计知识

**具体改动**：

#### 后端增强

1. **响应压缩 (gzip)**
   - 添加 compression 中间件
   - 压缩级别 6（平衡压缩率和性能）
   - 支持通过 `X-No-Compression` header 禁用压缩
   - 减少网络传输体积，提升加载速度

2. **细粒度限流策略**
   - 通用限流：100 次/分钟（所有 API）
   - 认证接口限流：10 次/15 分钟（防止暴力破解）
   - 写入操作限流：20 次/分钟（POST/PUT/PATCH/DELETE）
   - 支持 Standard Headers 返回限流信息

3. **CORS 增强配置**
   - 支持指定域名白名单（localhost、GitHub Pages、Netlify）
   - 暴露 `X-Request-ID`、`X-RateLimit-*` 响应头
   - 启用 credentials 支持
   - 预检请求缓存 24 小时

4. **敏感操作审计日志**
   - 新增 `audit.js` 中间件
   - 记录认证、写入、删除等敏感操作
   - 记录未授权访问和限流触发
   - 记录慢查询（>5秒）
   - 提供 `/api/v1/audit/logs` 和 `/api/v1/audit/stats` 接口

5. **健康检查增强**
   - 显示已启用的安全功能状态
   - 包含 compression、rateLimit、security 字段

#### 前端增强

6. **API Client 类**
   - Token 自动管理（保存、刷新、清除）
   - Token 过期前自动刷新（提前 5 分钟）
   - 401 错误自动重试
   - 429 限流等待后自动重试
   - 5xx 服务器错误自动重试（最多 3 次）
   - 请求超时控制（15 秒）
   - 网络断开检测
   - 便捷的 GET/POST/PUT/DELETE 方法

### 技术细节

- **压缩算法**：gzip，压缩级别 6
- **限流算法**：滑动窗口 + IP 识别
- **审计日志**：内存存储（最多 500 条），支持按操作类型、时间范围筛选
- **重试策略**：指数退避（1s、2s、3s）
- **Token 策略**：双 Token（Access + Refresh），Access Token 1 小时有效期

### 文件变更

**后端**：
- `Agent-Street-Backend/package.json` - 添加 compression 依赖
- `Agent-Street-Backend/src/index.js` - CORS 增强、压缩、限流、审计中间件
- `Agent-Street-Backend/src/middleware/audit.js` - 新增审计日志中间件
- `Agent-Street-Backend/src/routes/audit.js` - 新增审计日志路由

**前端**：
- `Agent-Street/index.html` - API Client 类实现
- `index.html` - 同步更新

### 效果预期

- API 响应更快（gzip 压缩减少 60-80% 传输量）
- 更安全的 API 访问（限流防滥用、审计可追溯）
- 更稳定的连接（自动重试提升成功率）
- 更好的用户体验（Token 无感刷新）
- 便于问题排查（请求 ID 全链路追踪）

#### MarketListing 模型（市场挂单）
- `status + price` 复合索引：市场列表排序
- `sellerId + status` 复合索引：卖家+状态筛选

#### Transaction 模型（交易记录）
- `createdAt` 索引：时间范围查询
- `toAgentId + createdAt` 复合索引：用户交易历史

**优化原理**：
- 复合索引遵循最左前缀原则，支持多种查询组合
- 避免全表扫描，提升查询效率
- 索引覆盖（Covering Index）减少回表查询

## Day 16: API 性能监控与请求日志系统

### 新增功能

#### 1. 请求追踪系统
- 为每个请求生成唯一追踪 ID（8位 UUID）
- 通过 `X-Request-ID` Header 返回，便于日志关联
- 开发环境控制台实时显示请求信息

#### 2. 性能监控指标
| 指标 | 说明 |
|------|------|
| `total` | 总请求数 |
| `success` | 成功请求数 |
| `error` | 错误请求数 |
| `avgResponseTime` | 平均响应时间 |
| `min/max` | 最快/最慢响应 |
| `slowQueries` | 慢查询数量 |

#### 3. 慢查询警告
- 阈值：>1000ms 自动标记为慢查询
- 控制台输出警告：`🐌 [SLOW] GET /api/v1/agents - 1500.23ms [a1b2c3d4]`
- 日志中标记 `isSlow: true`

#### 4. 端点统计排行
- 自动收集每个 API 端点的访问数据
- 记录：请求次数、平均响应、最大响应
- 支持 Top 10 排行展示

#### 5. 请求日志记录
- 最近 100 条请求日志
- 包含：时间、方法、路径、状态码、响应时间、IP、User-Agent
- 固定大小队列，内存友好
