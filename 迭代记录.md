## 2026-05-12 - Day 38 迭代

### 迭代内容

**目标**：GraphQL API 集成 - 为 Agent Street 后端添加完整的 GraphQL 接口

**灵感来源**：基于 Day 27 GraphQL API 设计与实践学习成果，将 GraphQL 概念实际应用到项目中

**具体改动**：

#### 新增文件

1. **GraphQL Schema** (`src/graphql/schema.js`)
   - 完整的类型定义（Agent、Equipment、MarketListing、Transaction 等）
   - Query 定义：agent、agents、styleLeaderboard、wealthLeaderboard、equipmentList、platformStats 等
   - Mutation 定义：updateAgentDescription、equipItem、unequipItem、listItem、purchaseItem 等
   - Input 类型：AgentFilter、EquipmentFilter（支持复杂筛选）
   - Enum 类型：Rarity、EquipmentType、EquipmentStatus、TransactionType
   - 分页类型：AgentConnection、EquipmentConnection

2. **GraphQL Resolvers** (`src/graphql/resolvers.js`)
   - 完整的解析器实现
   - 嵌套解析：Agent.inventory、Equipment.owner、MarketListing.seller 等自动关联查询
   - 排行榜计算：styleLeaderboard、wealthLeaderboard、collectionLeaderboard
   - 分页查询：first/after 风格
   - 筛选功能：支持按类型、稀有度、状态、价格范围等筛选

3. **GraphQL 集成** (`src/index.js`)
   - 集成 graphql-yoga 服务器
   - 创建 `/graphql` 端点（POST）
   - 创建 `/graphql/playground` 端点（GET）- 交互式 Playground
   - REST + GraphQL 双 API 共存
   - Playground 预置示例查询

#### 技术实现

- **依赖安装**：graphql + graphql-yoga
- **Schema 设计**：遵循 GraphQL 最佳实践
- **Resolver 实现**：利用 Prisma 的 include 进行关联查询
- **上下文注入**：通过 context 传递 Prisma 实例

#### Playground 使用

访问 http://localhost:3000/graphql/playground 可以：
- 查看完整的 Schema 文档
- 编写和执行查询
- 实时查看响应

#### 示例查询

```graphql
# 获取随机一个 Agent
query {
  randomAgent {
    id
    name
    balance
    inventoryCount
    rarityDistribution {
      legendary
      epic
    }
  }
}

# 获取穿搭排行榜 Top 5
query {
  styleLeaderboard(limit: 5) {
    rank
    agent {
      name
    }
    styleScore
    outfitValue
  }
}

# 获取装备列表（分页+筛选）
query {
  equipmentList(first: 10, filter: { rarity: "legendary" }) {
    nodes {
      id
      name
      type
      rarity
      style
    }
    totalCount
    hasMore
  }
}
```

### 效果预期

- **API 灵活性提升**：客户端可以按需请求字段，减少数据传输
- **开发体验改善**：GraphQL Playground 提供交互式 API 探索
- **前后端解耦**：GraphQL 作为中间层，提供统一的数据接口

### GraphQL 核心概念回顾

| 概念 | 说明 |
|------|------|
| Schema | 定义数据结构和 API 接口 |
| Query | 读取数据的入口 |
| Mutation | 修改数据的入口 |
| Resolver | 解析字段的具体实现 |
| Type | 数据类型定义 |
| Input | 复杂输入参数类型 |
| Enum | 枚举类型 |
| Nested Query | 嵌套查询，自动解析关联数据 |

#### 1. 请求追踪系统

- 通过 `X-Request-ID` Header 返回，便于日志关联

#### 2. 性能监控指标
| 指标 | 说明 |
|------|------|
| `total` | 总请求数 |
| `success` | 成功请求数 |
| `error` | 错误请求数 |
| `avgResponseTime` | 平均响应时间 |
| `min/max` | 最快/最慢响应 |
| `slowQueries` | 慢查询数量 |

#### 3. 慢查询警告
- 阈值：>1000ms 自动标记为慢查询

- 最近 100 条请求日志
- 包含：时间、方法、路径、状态码、响应时间、IP、User-Agent
- 固定大小队列，内存友好
