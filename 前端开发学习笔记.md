## 一、Web 基础知识

### 1.1 网页的三大核心技术

Web 开发由三大部分组成，俗称"前端三件套"：

| 技术 | 作用 | 比喻 |
|------|------|------|
| **HTML** | 网页的结构和内容 | 像房子的框架 |
| **CSS** | 网页的样式和布局 | 像房子的装修 |
| **JavaScript** | 网页的交互和行为 | 像房子的智能系统 |

#### HTML（超文本标记语言）
- **核心概念**：用标签定义页面结构
- **常用标签**：
  - `<div>` - 容器
  - `<h1>-<h6>` - 标题
  - `<p>` - 段落
  - `<img>` - 图片
  - `<a>` - 链接
  - `<ul>/<ol>` - 列表
  - `<form>` - 表单

#### CSS（层叠样式表）
- **三种引入方式**：
  1. 内联样式：`<div style="color:red">`
  2. 内部样式：`<style> div { color: red; } </style>`
  3. 外部样式：`<link rel="stylesheet" href="style.css">`
  
- **布局技术**：
  - Flexbox（弹性盒子）
  - Grid（网格布局）
  - 定位（position）

#### JavaScript
- **核心功能**：
  - 操作 DOM（页面元素）
  - 响应用户交互
  - 发送网络请求
  - 数据处理

## 二、前端框架选择

### 2.1 主流框架对比

| 特性 | Vue.js | React | 原生 HTML/CSS/JS |
|------|--------|-------|------------------|
| **学习曲线** | 低（平缓） | 中高（JSX 较难） | 最低 |
| **国内生态** | 最好（55%市场份额） | 中等（35%） | 无 |
| **开发效率** | 高（组件化） | 高（组件化） | 中 |
| **适用场景** | 中小型项目、快速开发 | 大型应用、复杂交互 | 简单页面、学习入门 |
| **文件体积** | 较小 | 中等 | 最小 |

### 2.2 我的选择建议

**对于本次项目，建议使用纯静态页面（HTML + CSS + JavaScript）**

**理由**：
1. ✅ 学习成本低 - 从基础开始，理解原理
2. ✅ 部署简单 - 可以直接部署到 GitHub Pages
3. ✅ 性能最优 - 无框架开销
4. ✅ 适合当前需求 - 主要是数据展示，不需要复杂的用户交互
5. ✅ 可以调用外部 API 获取数据

**后续升级路径**：
- 掌握基础后，可以学习 Vue.js
- Vue 3 + Vite 是现代开发的黄金组合

## 三、API 调用基础

### 3.1 什么是 API

API（应用程序接口）是前后端通信的桥梁。前端通过 HTTP 请求获取数据。

### 3.2 Fetch API 使用方法

```javascript
// 基本语法
fetch(url, options)
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error('Error:', error));

// POST 请求示例
fetch('https://api.example.com/data', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer YOUR_TOKEN'
  },
  body: JSON.stringify({
    message: 'Hello'
  })
});
```

### 3.3 Coze API 调用示例

```javascript
// Coze API 端点
const API_URL = 'https://api.coze.cn/open_api/v2/chat';

// 发送消息到 Coze
async function sendMessage(message) {
  const response = await fetch(API_URL, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': 'Bearer YOUR_API_KEY'
    },
    body: JSON.stringify({
      bot_id: 'YOUR_BOT_ID',
      user: 'user_123',
      query: message,
      stream: false
    })
  });
  
  const data = await response.json();
  return data;
}
```

### 3.4 重要注意事项

⚠️ **API Key 安全**：
- 不要在前端代码中硬编码 API Key
- 使用环境变量管理敏感信息
- 生产环境建议通过后端代理调用

## 五、静态 vs 动态网页

### 5.1 核心区别

| 对比项 | 静态网页 | 动态网页 |
|--------|----------|----------|
| **内容生成** | 预先渲染，固定内容 | 实时生成，动态内容 |
| **技术栈** | HTML/CSS/JS | 前后端 + 数据库 |
| **性能** | 极快（CDN友好） | 依赖后端优化 |
| **功能** | 简单 | 复杂（用户交互、数据处理） |
| **维护成本** | 低 | 高 |
| **典型用例** | 博客、宣传页 | 社交平台、电商 |

### 5.2 混合方案（推荐）

现代前端开发常用"静态页面 + API调用"的方式：
- 页面本身是静态的（快速加载）
- 通过 JavaScript 调用 API 获取动态数据
- 结合了静态网页的性能和动态网页的灵活性
